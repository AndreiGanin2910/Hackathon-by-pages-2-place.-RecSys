# Hackathon-by-pages-2-place.-RecSys


# 📚 Book Recommendation System

> Гибридная рекомендательная система для книжного сервиса, балансирующая между точной персонализацией и жанровым разнообразием витрины.

Путь к данным и описанию задачи: https://ods.ai/competitions/nto25-26-hack/dataset

---

## 🎯 Описание задачи

Книжный сервис хочет показывать пользователю **витрину рекомендаций на следующий месяц** (30 дней). Витрина должна решать две задачи одновременно:

| Цель | Описание |
|------|----------|
| **Релевантность** | Максимизировать шанс будущих взаимодействий (чтение или добавление в вишлист) |
| **Разнообразие** | Расширять кругозор пользователя, предлагая контент из разных жанров |

**Формально:** для каждого пользователя из тестовой выборки необходимо сформировать **ранжированный список из 20 изданий** (`edition_id`) из пула 200 кандидатов, где `rank = 1` — верхняя позиция витрины.

---

## 🏗 Архитектура решения

Решение представляет собой многоступенчатый пайплайн, объединяющий коллаборативную фильтрацию, контентный анализ, градиентный бустинг и diversity-aware постобработку.

### Коллаборативная фильтрация (6 моделей)

Каждая CF-модель привносит уникальный «взгляд» на данные, что усиливает ансамбль:

| Модель | Факторы | Особенность | Зачем нужна |
|--------|---------|-------------|-------------|
| **ALS** | 96 | Основная, веса read=3.0+0.1×rating | Базовый CF-сигнал |
| **ALS-Recent** | 96 | Только последние 90 дней | Адаптация к смене интересов |
| **ALS-2** | 48 | Сильнее регуляризация (0.1), seed=123 | Другой взгляд на паттерны |
| **ALS-TimedDecay** | 64 | Экспоненциальный decay (half-life=60д) | Плавное затухание старых данных |
| **BPR** | 64 | Попарное ранжирование (AUC-loss) | Оптимизирует порядок, а не ошибку |
| **SVD-CF** | 32 | sklearn TruncatedSVD | Прямое разложение (не итеративное) |

### Контентные сигналы

- **TF-IDF + TruncatedSVD** (dim=24): эмбеддинги из `название + описание + автор`, 35K фичей, uni+bigrams
- **Item2Vec** (Word2Vec, dim=64): обучен на хронологических последовательностях взаимодействий каждого пользователя (Skip-gram, window=10)

### Модели ранжирования (ансамбль из 8 моделей)

| Модель | Loss | Итерации | Назначение |
|--------|------|----------|------------|
| **CatBoostRanker** ×4 | YetiRank | 8000 | Listwise-ранжирование |
| **CatBoostClassifier** ×4 | MultiClass (3 класса) | 3000 | Вероятности P(read), P(wish), P(nothing) |

Каждая модель обучена на своём временном окне (сдвиги 10/15/20/25 дней назад). Предсказания усредняются.

### Постобработка

1. **Калибровка вероятностей**: temperature scaling (T=0.9) + per-user scaling до целевой суммы p_rel по top-50
2. **Diversity-aware reranking**: жадный MMR-подобный алгоритм с адаптивным переключением head→tail

## 📐 Схема обучения

Используется **5 временных окон** (cutoffs), сдвинутых назад от максимальной даты в данных:

```
Timeline:
──────────────────────────────────────────────────────────►
        │         │       │      │     │    │
     -25d      -20d    -15d   -10d   -7d  max_ts
        │         │       │      │     │
     train₄   train₃  train₂  train₁  val
        │         │       │      │     │
        └─ past ──┘       │      │     └── past → features
           future (30d) ──┘      │         future (30d) → labels
                                 │
                              ... и так для каждого окна
```

Для каждого окна:
- **past** = все взаимодействия до cutoff → используются для генерации признаков
- **future** = взаимодействия в `[cutoff, cutoff + 30 дней)` → метки
- **label** = `3` (read), `1` (wish), `0` (нет события)

Первое окно (7 дней назад) — **валидация** с early stopping, остальные 4 — **обучение**.

## 📏 Метрика оценивания

Итоговый скор — **комбинация релевантности и жанрового разнообразия**:

```
Score = mean(NDCG@20) × mean(Diversity@20)
```

### NDCG@20 (релевантность)

Сравнивает ранжированный список с фактическими будущими взаимодействиями:
- `rel = 3` — пользователь прочитал книгу (read)
- `rel = 1` — добавил в вишлист (wishlist)
- `rel = 0` — нет событий

### Diversity@20 (разнообразие)

Смесь двух компонентов (λ=0.5):

| Компонент | Что измеряет |
|-----------|-------------|
| **Genre Coverage** | Покрытие уникальных жанров среди релевантных рекомендаций (с позиционным дисконтом) |
| **Intra-List Diversity (ILD)** | Средняя Jaccard-distance между парами релевантных рекомендаций |

> Вклад в diversity учитывается **только для изданий с фактическими взаимодействиями** в тестовом окне.

## 📊 Результаты

### Формат submission

```csv
user_id,edition_id,rank
12345,67890,1
12345,11111,2
...
12345,99999,20
```

- `rank` — целое число от 1 до 20 (1 = верх витрины)
- Ровно **20 строк** на каждого пользователя
- Все `edition_id` из пула 200 кандидатов данного пользователя

### Diversity-aware Reranking

Алгоритм postprocessing'а — **жадный MMR-подобный** с двухфазной стратегией:

| Фаза | Позиции | Стратегия |
|------|---------|-----------|
| **Head** | 1–8 | Жадный выбор по base_score (максимальная релевантность) |
| **Tail** | 9–20 | Адаптивный баланс nDCG + Coverage + ILD |

Дополнительные ограничения:
- ≤3 книги одного автора в head, ≤1 в tail
- Запрет дубликатов одной книги (разные edition_id одного book_id)
- Адаптивный lock: если gap между позициями 8 и 9 мал (<0.05), диверсификация начинается раньше

## 🔑 Ключевые идеи

1. **Ансамбль разнородных CF-моделей**: 6 моделей с разными loss-функциями, гиперпараметрами и подвыборками данных дают устойчивый сигнал
2. **Multi-window обучение**: 4 временных сдвига для обучения снижают переобучение на конкретный момент времени
3. **Двойная голова (Ranker + Classifier)**: ранкер оптимизирует порядок, классификатор даёт калиброванные вероятности для reranking'а
4. **Калибровка per-user**: нормализация вероятностей под каждого пользователя для корректной работы MMR
5. **Метрика-aware reranking**: прямая оптимизация комбинации nDCG × Diversity через coverage + ILD компоненты
